VAPT on Zero Bank Simulation

Executive Summary

A security assessment of the internal corporate network of Zero Bank was performed on September 9, 2025. The penetration test simulated an attack from an external threat actor to discover and identify vulnerabilities in the infrastructure. A total of 20 vulnerabilities were identified, broken down by severity: 2 Critical, 6 High, 8 Medium, and 4 Low.

The highest severity vulnerabilities gave potential attackers the ability to intercept network requests, capture authentication hashes, and perform SMB relay attacks to compromise Active Directory. Exploiting these weaknesses could lead to privilege escalation to the domain-admin level and unrestricted access to sensitive data.

Project Scope

The engagement covered multiple facets of the organization's infrastructure to ensure a holistic security review. The in-scope items were:

  • Zero Bank’s internal network and Active Directory
  • Windows client systems
  • Application server (FTP and web services: VSFTP, DVWA)
  • Mutillidae web application and its database system

Testing Methodology

The testing methodology was divided into five distinct phases: Scope Definition, Reconnaissance, Vulnerability Analysis, Exploitation, and Reporting. The process involved both automated tools and manual exploitation techniques, including:

  • Reconnaissance: Performed port scanning, service enumeration, and OS fingerprinting to map potential attack surfaces.
  • Vulnerability Analysis: Matched identified services to known vulnerabilities such as the VSFTPD backdoor, SMB relay, and SQL injection.
  • Exploitation: Executed attacks like LLMNR/NBT-NS poisoning to capture NTLMv2 hashes, exploited the VSFTPD backdoor for shell access, and performed SQL injection on the Mutillidae database.
  • Post-Exploitation: Demonstrated privilege escalation, simulated lateral movement, and evaluated risks of data exfiltration.

Key Findings

MS17-010 (EternalBlue)

A critical Windows vulnerability allowing unauthenticated remote code execution, which could lead to rapid network-wide compromise.

VSFTPD v2.3.4 Backdoor

An insecure configuration in the FTP service that granted attackers an unauthorized remote shell on the host system without authentication.

SQL Injection

Improper input validation in web applications allowed for malicious SQL queries to bypass authentication and extract sensitive database information.

Cross-Site Scripting (XSS)

The application did not properly sanitize user-supplied input, allowing attackers to inject malicious scripts to hijack user sessions or steal cookies.
